# Соответствие раздела «Администраторы и менеджеры» ТЗ

| Требование ТЗ | Реализация |
|---------------|------------|
| Создание учётных записей администраторов и менеджеров | Форма «Добавить сотрудника», назначение роли admin/manager |
| Назначение ролей и уровней доступа | Выбор роли «Администратор платформы» / «Менеджер платформы»; описание ролей из БД (`roles.description`) и пояснение на странице списка. Детальная матрица прав по модулям — задел под расширение (см. `config/roles.php`, при необходимости — пакет прав). |
| Контроль активности и полномочий | В списке: статус (активен/заблокирован), дата последнего входа; на форме редактирования — блок «Контроль учётной записи» |
| Временная блокировка или отзыв доступа | **Блокировка:** действие «замок» → модальное подтверждение → `is_active = false`, вход запрещён. **Отзыв:** удаление ролей staff / удаление пользователя при отсутствии других ролей (как ранее) |
| Список: имя и контактные данные | Имя, email (колонка «Контакты») |
| Список: роль | Бейдж по роли admin/manager |
| Список: статус активен/заблокирован | Поле `users.is_active`, фильтр в списке |
| Список: дата последнего входа | Поле `users.last_login_at`, обновляется при успешном входе |
| Изменение ролей | Редактирование записи, смена `role_id` |
| Временно заблокировать доступ | `POST …/suspend` |
| Полностью отключить учётную запись | Отзыв ролей / удаление записи при отсутствии других ролей (`destroy`) |
| Различия администратора и менеджера | Тексты из сидера ролей + информационный блок на странице списка |

**Ограничения:** учётная запись с `id = 1` (главный администратор) не редактируется, не удаляется и не блокируется. Пользователь не может заблокировать или удалить сам себя.

**Миграция:** `2026_03_19_000001_add_is_active_and_last_login_to_users_table.php`

**Middleware:** `user.active` — заблокированный пользователь выходит из сессии при обращении к защищённым маршрутам.
