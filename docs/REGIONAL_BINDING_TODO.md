# Региональная привязка — нереализованный функционал

Список того, что нужно доделать по региональной привязке, когда появится остальной функционал (кабинет конечной компании, заказы, филиалы и т.д.). Уже готовые scope и методы перечислены в конце — их нужно только подключить.

---

## 1. Регион пользователя и филиалы

**ТЗ:** у каждого пользователя строго определённый регион работы; для конечной компании регион задаётся выбранным филиалом.

**Сделать:**
- [ ] Модель филиала (Branch) или профиль конечной компании с полем `region_id`.
- [ ] Связь: компания (конечная) → филиалы → регион (или компания → один регион, если без филиалов).
- [ ] Хранение в сессии текущего филиала/региона для конечной компании (например, `current_branch_id` или `current_region_id`).
- [ ] При выборе филиала — обновлять сессию и перезагружать каталог/данные по новому региону.

**Зависимости:** без этого кабинет конечной компании не сможет фильтровать по региону.

---

## 2. Кабинет конечной компании

**ТЗ:** каталог и заказы для роли «Конечная компания» с учётом региона (филиала).

**Сделать:**
- [ ] Маршруты и контроллеры для роли `end_company` (каталог, карточка товара, корзина/заказы — по мере появления модулей).
- [ ] Определение текущего региона: из выбранного филиала (или из профиля компании, если филиалов нет).
- [ ] В списке/каталоге товаров использовать `Product::availableInRegion($currentRegionId)`.
- [ ] В карточке товара подставлять цену через `$product->getPriceForRegion($currentRegionId)` и остатки через `$product->getAvailableStockInRegion($currentRegionId)`.
- [ ] Переключение филиала в шапке/профиле — обновление сессии и редирект или перезагрузка каталога.

**Использовать:** `Product::scopeAvailableInRegion()`, `getPriceForRegion()`, `getAvailableStockInRegion()` (см. раздел «Готовый код» ниже).

---

## 3. Кабинет дистрибьютора

**ТЗ:** дистрибьютор управляет регионами работы и видит ассортимент по регионам.

**Сделать:**
- [ ] Решить: отдельная модель/профиль дистрибьютора или общий с производителем (например, один профиль с регионами для ролей manufacturer/distributor).
- [ ] Если отдельный профиль — таблица/модель с привязкой к регионам (аналог `manufacturer_region`).
- [ ] Маршруты и экраны кабинета дистрибьютора: настройка регионов, просмотр ассортимента по регионам (при наличии своего каталога/товаров).

**Зависимости:** от выбора архитектуры (один профиль на обе роли или два).

---

## 4. Фильтрация каталога по региону (для конечной компании)

**ТЗ:** показывать только товары, доступные в регионе пользователя, с корректными остатками и ценами по региону.

**Сделать:**
- [ ] В контроллере каталога конечной компании получать `$currentRegionId` (из филиала/профиля/сессии).
- [ ] Запрос товаров: `Product::query()->availableInRegion($currentRegionId)->published()->...` (и остальные фильтры по категории, поиску и т.д.).
- [ ] В шаблонах каталога и карточки товара выводить цену и остатки только через `getPriceForRegion($currentRegionId)` и `getAvailableStockInRegion($currentRegionId)`.
- [ ] Не показывать цену/остатки/кнопку заказа для товаров, не входящих в `availableInRegion` (если такие вдруг попадут в выборку).

**Использовать:** `Product::scopeAvailableInRegion()`, `getPriceForRegion()`, `getAvailableStockInRegion()`.

---

## 5. Список поставщиков по региону

**ТЗ:** на карточке товара конечная компания видит только поставщиков (дистрибьюторов), обслуживающих её регион.

**Сделать:**
- [ ] На странице карточки товара для конечной компании: список поставщиков фильтровать по региону.
- [ ] Запрос: поставщики товара (производители с этим товаром), у которых есть регион: `ManufacturerProfile::inRegion($currentRegionId)` (или аналог для дистрибьютора).
- [ ] Учитывать наличие остатков в регионе: при необходимости фильтровать или помечать «Нет в наличии» поставщиков без остатков на складах в этом регионе.
- [ ] При появлении API «поставщики для моего региона» — использовать `ManufacturerProfile::scopeInRegion($regionId)`.

**Использовать:** `ManufacturerProfile::scopeInRegion()` (уже есть).

---

## 6. Товар «недоступен в вашем регионе»

**ТЗ:** товар без поставщика в регионе по умолчанию скрыт или с пометкой «Недоступно в вашем регионе», без цены/остатков и без кнопки «Добавить в заказ».

**Сделать:**
- [ ] В каталоге конечной компании по умолчанию не показывать товары, не входящие в `availableInRegion($currentRegionId)` (уже обеспечивается, если каталог строится через `availableInRegion`).
- [ ] Опционально: настройка «показывать недоступные с пометкой» — тогда выводить товары с подписью «Недоступно в вашем регионе», без цены, остатков и кнопки заказа.
- [ ] В шаблоне карточки: если товар не доступен в текущем регионе — показывать блок «Недоступно в вашем регионе» и скрывать кнопку заказа.

---

## 7. Аналоги и регион

**ТЗ:** аналоги, поставляемые только вне текущего региона, не показывать или маркировать «Недоступно в вашем регионе».

**Сделать:**
- [ ] Реализовать сущность «аналоги» (связь товар ↔ товар или отдельная таблица).
- [ ] При выводе блока аналогов на карточке товара для конечной компании фильтровать аналоги по региону: показывать только те, что `availableInRegion($currentRegionId)`.
- [ ] Опционально: недоступные в регионе аналоги показывать с пометкой «Недоступно в вашем регионе» без кнопки заказа.

**Зависимости:** появление модуля аналогов.

---

## 8. Заказы и проверка региона

**ТЗ:** нельзя оформить заказ у поставщика, не работающего в регионе пользователя.

**Сделать:**
- [ ] При создании/редактировании заказа (выбор поставщика или добавление позиции от поставщика) проверять: поставщик входит в список для текущего региона пользователя.
- [ ] Проверка: `ManufacturerProfile::inRegion($userRegionId)->where('id', $selectedSupplierId)->exists()` (или аналог для дистрибьютора); при неуспехе — ошибка валидации или 403.
- [ ] В API/формах не допускать подстановку поставщика не из региона (например, скрывать таких в выпадающем списке или блокировать отправку).

**Использовать:** `ManufacturerProfile::scopeInRegion()`.

**Зависимости:** появление модуля заказов.

---

## 9. Смена региона (филиала) и обновление ассортимента

**ТЗ:** при смене филиала/региона — полное обновление товаров, поставщиков, остатков, цен, скидок, аналогов.

**Сделать:**
- [ ] В интерфейсе: переключатель филиала (или региона) в шапке/профиле для конечной компании.
- [ ] По смене: обновить сессию (текущий филиал/регион), затем редирект на каталог или текущую страницу — данные подгрузятся заново с новым регионом.
- [ ] Все запросы каталога, карточки товара, списка поставщиков и заказа должны опираться на актуальный `current_region_id` из сессии/филиала.

**Зависимости:** п. 1 (филиалы/регион в сессии).

---

## 10. Ограничения прав на регион

**ТЗ:** пользователь не может вручную менять регион при фиксированном регионе компании; региональные настройки не меняют без прав.

**Сделать:**
- [ ] После появления филиалов/профиля конечной компании: политики (Policy) или middleware — кто может менять текущий филиал/регион.
- [ ] Ограничение смены региона: только при наличии права (например, «управление филиалами» или «настройки компании»); при фиксированном одном регионе — скрыть переключатель.
- [ ] Региональные настройки у поставщика (производитель/дистрибьютор): проверка прав на редактирование профиля/регионов.

**Зависимости:** п. 1–3 (филиалы, кабинеты).

## Готовый код (уже есть в проекте)

Использовать при реализации пунктов выше:

| Что нужно | Где лежит |
|-----------|-----------|
| Товары, доступные в регионе (по привязке ассортимента) | `Product::scopeForRegion($query, $regionId)` |
| Товары с поставщиком в регионе + привязка товара к региону | `Product::scopeAvailableInRegion($query, $regionId)` |
| Цена для региона (региональная или базовая) | `Product::getPriceForRegion(?int $regionId)` |
| Остатки только по складам региона | `Product::getAvailableStockInRegion(?int $regionId)` |
| Поставщики, работающие в регионе | `ManufacturerProfile::scopeInRegion($query, $regionId)` |

Подробная проверка по ТЗ: `docs/REGIONAL_BINDING_TZ_CHECK.md`.
